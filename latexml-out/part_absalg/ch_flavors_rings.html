Prototypical example for this section: ‚Ñ§ and polynomial rings in general.

5.7 Extra: Euclidean domains

This chapter will not be used later on, but it is of historical interest.

Recall that a PID is a ring where you can take the gcd of any family of elements.

We all know that the most popular algorithm to compute the gcd of two elements in ‚Ñ§ is the Euclidean algorithm:

  • ‚Ä¢

    Start with two integers a and b.

  • ‚Ä¢

    If either of a and b is 0, we‚Äôre done. The gcd is the nonzero element.

  • ‚Ä¢

    Otherwise, assume |a|‚â•|b|, divide a by b to get some remainder r such that |r|<|b|, replace a with r, and continue the algorithm.

This algorithm is very efficient ‚Äî it can be proven that the algorithm only takes logarithmically many steps in the size of a and b ‚Äî for instance, if a and b are on the order of 10100, at worst 500 steps are needed.

Naturally, the following questions are raised:

On which rings can we perform the same algorithm?

We will see that we can in fact do it on several rings! For example, the ring of Gaussian integers ‚Ñ§‚Å¢[i], the Eisenstein integers ‚Ñ§‚Å¢[œâ], and so on.

If we look at the algorithm description above, what makes the algorithm work? It‚Äôs the absolute value |‚ãÖ| which is used to compare the magnitude of two numbers, and this absolute value satisfies two conditions:

  • ‚Ä¢

    It outputs nonnegative integer values ‚Äî that way, the algorithm will eventually terminate.

  • ‚Ä¢

    For any two ring elements a and b, where b‚â†0, there exist some q such that r=a‚àíq‚Å¢b has smaller absolute value than b.
