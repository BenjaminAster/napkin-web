Chapter 93 Forcing

We are now going to introduce Paul Cohen’s technique of forcing, which we then use to break the Continuum Hypothesis.

Here is how it works. Given a transitive model M and a poset ℙ inside it, we can consider a “generic” subset G⊆ℙ, where G is not in M. Then, we are going to construct a bigger universe M⁢[G] which contains both M and G. (This notation is deliberately the same as ℤ⁢[2], for example – in the algebra case, we are taking ℤ and adding in a new element 2, plus everything that can be generated from it.) By choosing ℙ well, we can cause M⁢[G] to have desirable properties.

The models M and M⁢[G] will share the same ordinals, which is represented here as M being no taller than M⁢[G]. But one issue with this is that forcing may introduce some new bijections between cardinals of M that were not there originally; this leads to the phenomenon called cardinal collapse: quite literally, cardinals in M will no longer be cardinals in M⁢[G], and instead just an ordinal. This is because in the process of adjoining G, we may accidentally pick up some bijections which were not in the earlier universe. In the diagram drawn, this is the function f mapping ω to ℵ1M. Essentially, the difficulty is that “κ is a cardinal” is a Π1 statement.

In the case of the Continuum Hypothesis, we’ll introduce a ℙ such that any generic subset G will “encode” ℵ2M real numbers. We’ll then show cardinal collapse does not occur, meaning ℵ2M⁢[G]=ℵ2M. Thus M⁢[G] will have ℵ2M⁢[G] real numbers, as desired.

93.1 Setting up posets

Prototypical example for this section: Infinite Binary Tree

Let M be a transitive model of 𝖹𝖥𝖢. Let ℙ=(ℙ,≤)∈M be a poset with a maximum element 1ℙ which lives inside a model M. The elements of ℙ are called conditions; because they will force things to be true in M⁢[G].

Definition 93.1.1.

A subset D⊆ℙ is dense if for all p∈ℙ, there exists a q∈D such that q≤p.

Examples of dense subsets include the entire ℙ as well as any downwards “slice”.

Definition 93.1.2.

For p,q∈ℙ we write p∥q, saying “p is compatible with q”, if there exists r∈ℙ with r≤p and r≤q. Otherwise, we say p and q are incompatible and write p⟂q.

  • •

Now, I can specify what it means to be “generic”.

Definition 93.1.5.

A nonempty set G⊆ℙ is a filter if

  1. (a)

    The set G is upwards-closed: ∀p∈G⁢(∀q≥p)⁢(q∈G).

  2. (b)

    Any pair of elements in G is compatible.

We say G is M-generic if for all D which are in the model M, if D is dense then G∩D≠∅.

Question 93.1.6

Show that if G is a filter then 1ℙ∈G.

Note that the condition that D∈M is important, because:

Question 93.1.7

On the infinite binary tree, show that:

  • •

    For every filter G, there’s a dense subset D (not necessarily in M) such that G∩D=∅.

  • •

    Specifically, if G∈M, then such a set D can be chosen such that D∈M — in particular, G is not M-generic.

We will formalize this later in Lemma 93.2.5.

Example 93.1.8 (Generic filters on the infinite binary tree)

Let ℙ=2<ω. The generic filters on ℙ are sets of the form

{0,b1,b1⁢b2,b1⁢b2⁢b3,…}.

So every generic filter on ℙ corresponds to a binary number b=0.b1⁢b2⁢b3⁢….111Note that it may be the case that two distinct filters correspond to the same real number, such as 0.1000⁢… and 0.0111⁢…, but such filters are necessarily not generic.

It is harder to describe which reals correspond to generic filters, but they should really “look random”. For example, the set of strings ending in 011 is dense, so one should expect “011” to appear inside b, and more generally that b should contain every binary string. So one would expect the binary expansion of π−3 might correspond to a generic, but not something like 0.010101⁢…. That’s why we call them “generic”.

Example 93.1.9 (Generic filters on the infinite chain)

There’s only one generic filter on ℙ=(ℕ,≥): ℕ itself.

This is indeed generic — it hits every dense set. This doesn’t “look random” by any measure, you may say — but if you think about it, if you start at the root 1ℙ=1 and move down randomly at each step, there’s only one choice where to go!

Notice that if p≥q, then the sentence q∈G tells us more information than the sentence p∈G. In that sense q is a stronger condition. In another sense 1ℙ is the weakest possible condition, because it tells us nothing about G; we always have 1ℙ∈G since G is upwards closed.

93.2 More properties of posets

We had better make sure that generic filters exist.

Example 93.2.1

If ℙ is the infinite binary tree and M contains every subset of ℙ, then generic filter does not exist — for every filter G⊆ℙ, ℙ∖G is a dense set D∈M, and G∩M=∅.

In fact this is kind of tricky, but for countable models it works:

Lemma 93.2.2 (Rasiowa-Sikorski lemma)

Suppose M is a countable transitive model of 𝖹𝖥𝖢 and ℙ is a partial order. Then there exists an M-generic filter G.

Proof.

Essentially, hit them one by one. 93B. ∎

Fortunately, for breaking 𝖢𝖧 we would want M to be countable anyways.

The other thing we want to do to make sure we’re on the right track is guarantee that a generic set G is not actually in M. (Analogy: ℤ⁢[3] is a really stupid extension.) The condition that guarantees this is:

Definition 93.2.3.

A partial order ℙ is splitting if for all p∈ℙ, there exists q,r≤p such that q⟂r.

Example 93.2.4 (Infinite binary tree is (very) splitting)

The infinite binary tree is about as splitting as you can get. Given p∈2<ω, just consider the two elements right under it.

Lemma 93.2.5 (Splitting posets omit generic sets)

Suppose ℙ is splitting. Then if F⊆ℙ is a filter such that F∈M, then ℙ∖F is dense. In particular, if G⊆ℙ is generic, then G∉M.

Proof.

Consider p∉ℙ∖F⇔p∈F. Since ℙ is splitting, there exist q,r≤p which are not compatible. Since F is a filter it cannot contain both; we must have one of them outside F, say q. Hence every element of p∈ℙ∖(ℙ∖F) has an element q≤p in ℙ∖F. That’s enough to prove ℙ∖F is dense.

Question 93.2.6

Deduce the last assertion of the lemma about generic G. ∎

93.3 Names, and the generic extension

We now define the names associated to a poset ℙ.

Definition 93.3.1.

Suppose M is a transitive model of 𝖹𝖥𝖢, ℙ=(ℙ,≤)∈M is a partial order. We define the hierarchy of ℙ-names recursively by

Name0 =∅
Nameα+1 =𝒫⁢(Nameα×ℙ)
Nameλ =⋃α<λNameα.

Finally, Name=⋃αNameα denote the class of all ℙ-names.

(These Nameα’s are the analog of the Vα’s: each Nameα is just the set of all names with rank ≤α.)

Definition 93.3.2.

For a filter G, we define the interpretation of τ by G, denoted τG, using the transfinite recursion

τG={σG∣⟨σ,p⟩∈τ⁢ and ⁢p∈G}.

We then define the model

M⁢[G]={τG∣τ∈NameM}

where NameM are the elements of the class Name that belongs to M. Thus M⁢[G] is a set. In words, M⁢[G] is the interpretation of all the possible ℙ-names (as computed by M).

You should think of a ℙ-name as a “fuzzy set”. Here’s the idea. Ordinary sets are collections of ordinary sets, so fuzzy sets should be collections of fuzzy sets. These fuzzy sets can be thought of like the Ghosts of Christmases yet to come: they represent things that might be, rather than things that are certain. In other words, they represent the possible futures of M⁢[G] for various choices of G.

Every fuzzy set has an element p∈ℙ pinned to it. When it comes time to pass judgment, we pick a generic G and filter through the universe of ℙ-names. The fuzzy sets with an element of G attached to it materialize into the real world, while the fuzzy sets with elements outside of G fade from existence. The result is M⁢[G].

Example 93.3.3 (First few levels of the name hierarchy)

Let us compute

Name0 =∅
Name1 =𝒫⁢(∅×ℙ)
={∅}
Name2 =𝒫⁢({∅}×ℙ)
=𝒫⁢({⟨∅,p⟩∣p∈ℙ}).

Compare the corresponding von Neumann universe.

V0=∅,V1={∅},V2={∅,{∅}}.
Example 93.3.4 (Example of an interpretation)

As we said earlier, Name1={∅}. Now suppose

τ={⟨∅,p1⟩,⟨∅,p2⟩,…,⟨∅,pn⟩}∈Name2.

Then

τG={∅∣⟨∅,p⟩∈τ⁢ and ⁢p∈G}={{∅}if some ⁢pi∈G∅otherwise.

In particular, since 1ℙ∈G, then:

  • •

    when n=0, τ=∅, so τG=∅.

  • •

    when n=1 and p1=1ℙ, τ={⟨∅,1ℙ⟩}, so τG={∅}.

So,

{τG∣τ∈Name2}=V2.

In fact, this holds for any natural number n, not just 2.

So, M⁢[G] and M agree on finite sets.

Now, we want to make sure M⁢[G] contains the elements of M. The proof of {τG∣τ∈Name2}=V2 above can be easily adapted: Since 1ℙ must be in G, we define for every x∈M the set

xˇ={⟨yˇ,1ℙ⟩∣y∈x}

by transfinite recursion. Basically, xˇ is just a copy of x where we tag every element at every nesting level with 1ℙ.

Example 93.3.5

Compute 0ˇ=0 and 1ˇ={⟨0ˇ,1ℙ⟩}. Thus

(0ˇ)G=0and(1ˇ)G=1.
Question 93.3.6

Show that in general, (xˇ)G=x. (Rank induction.)

However, we’d also like to cause G to be in M⁢[G]. In fact, we can write down the name exactly: we define

ℙ˙≔{⟨pˇ,p⟩∣p∈ℙ}.
Question 93.3.7

Show that (ℙ˙)∈NameM, and (ℙ˙)G=G.

Question 93.3.8

Verify that M⁢[G] is transitive: that is, if σG∈τG∈M⁢[G], show that σG∈M⁢[G]. (This is offensively easy.)

In summary,

Moral 93.3.9 —

M⁢[G] is a transitive model extending M (it contains G).

Moreover, it is reasonably well-behaved even if G is just a filter. Let’s see what we can get off the bat.

Lemma 93.3.10 (Properties obtained from filters)

Let M be a transitive model of 𝖹𝖥𝖢. If G is a filter, then M⁢[G] is transitive and satisfies Extensionality, Foundation, EmptySet, Infinity, Pairing, and Union.

This leaves PowerSet, Replacement, and Choice.

Proof.

We get Extensionality and Foundation for free. Then Infinity and EmptySet follows from M⊆M⁢[G].

For Pairing, suppose σ1G,σ2G∈M⁢[G]. Then

σ={⟨σ1,1ℙ⟩,⟨σ2,1ℙ⟩}

satisfies σG={σ1G,σ2G}. (Note that we used M⊨Pairing.) Union is left as a problem, which you are encouraged to try now. ∎

Up to here, we don’t need to know anything about when a sentence is true in M⁢[G]; all we had to do was contrive some names like xˇ or {⟨σ1,1ℙ⟩,⟨σ2,1ℙ⟩} to get the facts we wanted. But for the remaining axioms, we are going to need this extra power. For this, we have to introduce the fundamental theorem of forcing.

93.4 Fundamental theorem of forcing

The model M unfortunately has no idea what G might be, only that it is some generic filter.222You might say this is a good thing; here’s why. We’re trying to show that ¬𝖢𝖧 is consistent with 𝖹𝖥𝖢, and we’ve started with a model M of the real universe V. But for all we know 𝖢𝖧 might be true in V (what if V=L?), in which case it would also be true of M. Nonetheless, we boldly construct M⁢[G] an extension of the model M. In order for it to behave differently from M, it has to be out of reach of M. Conversely, if M could compute everything about M⁢[G], then M⁢[G] would have to conform to M’s beliefs. That’s why we worked so hard to make sure G∈M⁢[G] but G∉M. Nonetheless, we are going to define a relation ⊩, called the forcing relation. Roughly, we are going to write

p⊩φ⁢(σ1,…,σn)

where p∈ℙ, σ1,…,σn∈M⁢[G], if and only if:

For any generic G, if p∈G, then M⁢[G]⊨φ⁢[σ1G,…,σnG].

Note that ⊩ is defined without reference to G: it is something that M can see. We say p forces the sentence φ⁢(σ1,…,σn). And miraculously, we can define this relation in such a way that the converse is true: a sentence holds if and only if some p forces it.

Theorem 93.4.1 (Fundamental theorem of forcing)

Suppose M is a transitive model of ZF. Let ℙ∈M be a poset, and G⊆ℙ is an M-generic filter. Then,

  1. (1)

    Consider σ1,…,σn∈NameM. Then

    M⁢[G]⊨φ⁢[σ1G,…,σnG]

    if and only if there exists a condition p∈G such that p forces the sentence φ⁢(σ1,…,σn). We denote this by p⊩φ⁢(σ1,…,σn).

  2. (2)

    This forcing relation is (uniformly) definable in M.

I’ll tell you how the definition works in the next section.

93.5 (Optional) Defining the relation

Here’s how we’re going to go. We’ll define the most generous condition possible such that the forcing works in one direction (p⊩φ⁢(σ1,…,σn) means M⁢[G]⊨φ⁢[σ1G,…,σnG]). We will then cross our fingers that the converse also works.

We proceed by induction on the formula complexity. It turns out in this case that the atomic formulas (base cases) are hardest and themselves require induction on ranks.

For some motivation, let’s consider how we should define p⊩τ1∈τ2 assuming that we’ve already defined p⊩τ1=τ2. We need to ensure this holds iff

∀M-generic G with p∈G:M⁢[G]⊨τ1G∈τ2G.

So it suffices to ensure that any generic G∋p hits a condition q which forces τ1G to equal a member τG of τ2G. In other words, we want to choose the definition of p⊩τ1∈τ2 to hold if and only if

{q∈ℙ∣∃⟨τ,r⟩∈τ2⁢(q≤r∧q⊩(τ=τ1))}

is dense below in p. In other words, if the set is dense, then the generic must hit q, so it must hit r (recall that a filter is upwards-closed), meaning that ⟨τr⟩∈τ2 will get interpreted such that τG∈τ2G, and moreover the q∈G will force τ1=τ.

Now let’s write down the definition…In what follows, the ⊩ omits the M and ℙ.

Definition 93.5.1.

Let M be a countable transitive model of ZFC. Let ℙ∈M be a partial order. For p∈ℙ and φ⁢(σ1,…,σn) a formula in the language of set theory, we write τ⊩φ⁢(σ1,…,σn) to mean the following, defined by induction on formula complexity plus rank.

  1. (1)

    p⊩τ1=τ2 means

    1. (i)

      For all ⟨σ1,q1⟩∈τ1 the set

      Dσ1,q1≔{r∣r≤q1→∃⟨σ2,q2⟩∈τ2⁢(r≤q2∧r⊩(σ1=σ2))}.

      is dense in p. (This encodes “τ1⊆τ2”.)

    2. (ii)

      For all ⟨σ2,q2⟩∈τ2, the set Dσ2,q2 defined similarly is dense below p.

  2. (2)

    p⊩τ1∈τ2 means

    {q∈ℙ∣∃⟨τ,r⟩∈τ2⁢(q≤r∧q⊩(τ=τ1))}

    is dense below p.

  3. (3)

    p⊩φ∧ψ means p⊩φ and p⊩ψ.

  4. (4)

    p⊩¬φ means ∀q≤p, q⊮φ.

  5. (5)

    p⊩∃x⁢φ⁢(x,σ1,…,σn) means that the set

    {q∣∃τ⁢(q⊩φ⁢(τ,σ1,…,σn))}

    is dense below p.

This is definable in M! All we’ve referred to is ℙ and names, which are in M. (Note that being dense is definable.) Actually, in parts (3) through (5) of the definition above, we use induction on formula complexity. But in the atomic cases (1) and (2) we are doing induction on the ranks of the names.

So, the construction above gives us one direction (I’ve omitted tons of details, but…).

Now, how do we get the converse: that a sentence is true if and only if something forces it? Well, by induction, we can actually show:

Lemma 93.5.2 (Consistency and Persistence)

We have

  1. (1)

    (Consistency) If p⊩φ and q≤p then q⊩φ.

  2. (2)

    (Persistence) If {q∣q⊩φ} is dense below p then p⊩φ.

You can prove both of these by induction on formula complexity. From this we get:

Corollary 93.5.3 (Completeness)

The set {p∣p⊩φ⁢ or ⁢p⊩¬φ} is dense.

Proof.

We claim that whenever p⊮φ then for some p¯≤p we have p¯⊩¬φ; this will establish the corollary.

By the contrapositive of the previous lemma, {q∣q⊩φ} is not dense below p, meaning for some p¯≤p, every q≤p¯ gives q⊮φ. By the definition of p⊩¬φ, we have p¯⊩¬φ. ∎

And this gives the converse: the M-generic G has to hit some condition that passes judgment, one way or the other. This completes the proof of the fundamental theorem.

93.6 The remaining axioms

Theorem 93.6.1 (The generic extension satisfies ZFC)

Suppose M is a transitive model of 𝖹𝖥𝖢. Let ℙ∈M be a poset, and G⊆ℙ is an M-generic filter. Then

M⁢[G]⊨𝖹𝖥𝖢.
Proof.

We’ll just do Comprehension, as the other remaining axioms are similar.

Suppose σG,σ1G,…,σnG∈M⁢[G] are a set and parameters, and φ⁢(x,x1,…,xn) is a formula in the language of set theory. We want to show that the set

A={x∈σG∣M⁢[G]⊨φ⁢[x,σ1G,…,σnG]}

is in M⁢[G]; i.e. it is the interpretation of some name.

Note that every element of σG is of the form ρG for some ρ∈dom⁡(σ) (a bit of abuse of notation here, σ is a bunch of pairs of names and p’s, and the domain dom⁡(σ) is just the set of names). So by the fundamental theorem of forcing, we may write

A={ρG∣ρ∈dom⁡(σ)⁢ and ⁢∃p∈G⁢(p⊩ρ∈σ∧φ⁢(ρ,σ1,…,σn))}.

To show A∈M⁢[G] we have to write down a τ such that the name τG coincides with A. We claim that

τ={⟨ρ,p⟩∈dom⁡(σ)×ℙ∣p⊩ρ∈σ∧φ⁢(ρ,σ1,…,σn)}

is the correct choice. It’s actually clear that τG=A by construction; the “content” is showing that τ is in actually a name of M, which follows from M⊨Comprehension.

So really, the point of the fundamental theorem of forcing is just to let us write down this τ; it lets us show that τ is in NameM without actually referencing G. ∎

93.7 A few harder problems to think about

Problem 93A.

For a filter G and M a transitive model of 𝖹𝖥𝖢, show that M⁢[G]⊨Union.

Problem 93B (Rasiowa-Sikorski lemma).

Show that in a countable transitive model M of 𝖹𝖥𝖢, one can find an M-generic filter on any partial order. Let D1, D2, … be the dense sets (there are countably many of them). Since M is countable, there are only countably many dense sets (they live in M!), say

D1,D2,…,Dn,…∈M.

Using Choice, let p1∈D1, and then let p2≤p1 such that p2∈D2 (this is possible since D2 is dense), and so on. In this way we can inductively exhibit a chain

p1≥p2≥p3≥…

with pi∈Di for every i.

Hence, we want to generate a filter from the {pi}. Just take the upwards closure – let G be the set of q∈ℙ such that q≥pn for some n. By construction, G is a filter (this is actually trivial). Moreover, G intersects all the dense sets by construction.
